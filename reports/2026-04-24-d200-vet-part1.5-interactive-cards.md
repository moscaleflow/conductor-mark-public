# D200 Part 1.5 — Interactive Per-Issue Cards

**Date:** 2026-04-24
**Agent:** Coder-1 (3 sub-agents via Pattern 23 fan-out)
**Status:** COMMITTED (pending deploy)

## What shipped

Replaced wall-of-text contract analysis display with interactive per-issue cards. Each issue has Accept/Reject/Edit/Explain Further buttons. Two new API endpoints back the interactive features. Vet prompt trimmed to 3-line verdict since cards handle per-issue detail.

| A — Components | IssueCard extend, InlineRedlineEditor new, IssueListExpandable new, page.tsx cleanup | 4 files | ~4 min |
| B — API endpoints | refine-redline, explain-further | 2 files | ~1 min |
| C — Vet prompt | CONTRACT VET MODE section update | 1 file | ~30s |

All 3 agents ran in parallel. Zero integration conflicts.

### Components

**IssueCard.tsx** (extended):
- 4 action buttons: Accept (Check), Reject (X), Edit (Pencil), Explain Further (MessageCircle)
- Visual states: accepted (green left border + tint), rejected (red border + opacity 0.5 + strikethrough), edited (amber border + badge)
- Accept/Reject buttons hide after decision (only show when pending)
- Inline explanation box renders below actions when Explain Further returns

**InlineRedlineEditor.tsx** (new):
- Textarea pre-filled with `suggested_revision`
- "Ask Milo to refine" calls `/api/ask/refine-redline`
- Loading state, error display, cancel button

**IssueListExpandable.tsx** (new):
- Extracted from inline ContractAnalysisCard in page.tsx
- Per-issue state machine: pending -> accepted / rejected / edited
- State: expandedIssues, issueStates, editingIssueId, explanations, explanationLoading
- Action summary bar at bottom: "X accepted, Y rejected, Z edited"

**page.tsx** (cleaned up):
- Removed 170 lines of inline ContractAnalysisCard + IssueField
- Single import of IssueListExpandable

### API endpoints

**POST /api/ask/refine-redline**:
- Input: analysisId, issueId, currentText, editedText
- Calls Claude Sonnet 4.6 with redline-refinement system prompt
- Returns: `{ ok: true, revisedText }`

**POST /api/ask/explain-further**:
- Input: analysisId, issueId, issueTitle, clauseReference, problemExplanation
- Calls Claude Sonnet 4.6 with plain-English explanation prompt
- Returns: `{ ok: true, explanation }`

Both endpoints: maxDuration 30s, no DB writes, ANTHROPIC_API_KEY guard.

### Vet prompt update

CONTRACT VET MODE section changed from "output full RED FLAGS / GREEN FLAGS wall-of-text" to:
- One gut-read sentence
- CONFIDENCE RATING: X out of 5
- One recommendation sentence
- 3-4 lines max — the cards handle per-issue detail
