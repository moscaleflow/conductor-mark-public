## C2: Drafts Pill

**Design decision:** No dedicated drafts table exists. D86 notes "Drafts needs milo-outreach wire." Probed 8 candidate table names — all 404. The closest draft-like data: `send_log` WHERE status IN ('dry_run', 'blocked_manual').

**Rationale:** `outreach_templates` stores reusable templates (opener/followup1/followup2), not pending operator actions. `send_log` with blocked/dry_run status represents outreach messages Milo generated but held for human review (human-in-the-loop gate or test mode). When milo-outreach starts generating live messages, `blocked_manual` entries will surface here automatically.

**Pattern 17:** Schema from milo-outreach migration 00001. Status CHECK: ('sent', 'delivered', 'bounced', 'failed', 'dry_run', 'blocked_env', 'blocked_tenant', 'blocked_manual'). Columns verified: id, tenant_id, lead_id, sequence_position, channel, to_email, subject, status, gate_state, sent_at.

**Severity:** blocked_manual = warning (needs human action), dry_run = info (test mode).

**Config:** revenue_at_risk: 0, snoozable: false, action_label: "Review draft".

**Roles:** outreach, admin.

## C3: Publishers Pill

**Source:** `crm_counterparties` WHERE counterparty_type = 'publisher' AND tenant_id = 'tlp'. Excludes terminal statuses (active, paused, churned, blacklisted, archived).

**Pattern 17:** Schema from @milo/crm migration 00001. counterparty_type CHECK: ('publisher', 'buyer', 'vendor', 'partner', 'client', 'prospect'). Status CHECK: ('lead', 'contacted', 'qualified', 'onboarding', 'active', 'paused', 'churned', 'blacklisted', 'archived'). Columns verified: id, tenant_id, company, status, contact_name, contact_email, source, updated_at, created_at.

**Attention-only filter** (same pattern as Sales D143 C2):

| Status | Stall Threshold |
|--------|----------------|
| lead | 14 days |
| contacted | 14 days |
| qualified | 21 days |
| onboarding | 14 days |

Plus recently changed (updated_at within 48h).

**Architecture:** Shared `fetchCounterpartyPill(type)` helper. Publishers and Buyers are thin wrappers calling it with 'publisher' / 'buyer'.

**Severity:** 2x threshold = critical, 1x = warning, recently changed = info.

**Config:** revenue_at_risk: 0, snoozable: false, action_label: "Open publisher".

**Roles:** operations, admin.
