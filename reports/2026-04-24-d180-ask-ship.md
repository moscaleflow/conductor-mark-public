## Architecture

### /api/ask route (`src/app/api/ask/route.ts`)
- POST endpoint, SSE streaming response
- Model: claude-sonnet-4-6 with prompt caching (`cache_control: { type: 'ephemeral' }`)
- Tool: `web_search` (max 5 uses per request) for vet/sales/publisher research
- Rate limiting: IP-hashed, 30 requests/hour, 100/day via `ask_rate_limits` table
- Blacklist pre-check: on vet pill, scans blacklist table for entity name matches, prepends `[BLACKLIST HIT: ...]` to user message so the system prompt's pre-check handler activates
- Chat logging: every request logged to `chat_logs` with pill, input/output text, token usage, cache stats, latency, IP hash
- Max message length: 10,000 characters
- Max tokens: 2048
- Temperature: 0.4

### /ask page (`src/app/ask/page.tsx`)
- Empty state: centered "Ask Milo" with 4 pill cards in 2x2 grid
- Active state: header with pill tabs (switchable), message list, textarea input
- SSE consumption: real-time streaming with delta events
- File drop zone: drag-and-drop text files (max 100KB), content appended to message
- Stop button during streaming, back button to reset
- Usage bar: latency, token count, cache hit indicator
- Dark theme matching existing app (`#0a0a0f` background, indigo accents)

### System prompts (`src/lib/ask/prompts/`)
- `vet.ts` — Entity vetting with fraud-default posture. Locked output format (RED FLAGS, GREEN FLAGS, CONFIDENCE RATING X/5). Blacklist pre-check handler.
- `sales.ts` — Buyer prospecting + outreach drafting. Hormozi-influenced framework. Re-engagement rules.
- `publisher.ts` — Publisher recruitment + evaluation. Traffic source intelligence (SEO, PPC, social, radio/TV, warm transfer).
- `buyer.ts` — Account management, call prep, dispute handling, blacklist decisions. Billing type awareness (CPA/RTB/CPL/CPQL).

All prompts follow locked Milo voice: first-person ("I", "me", "Milo"), slightly cocky, direct, useful. Never "we", "our team", "our AI".

### @milo/ai-client v0.1.3 (vendored)
- `cacheSystemPrompt?: boolean` — wraps system in array format with `cache_control: { type: 'ephemeral' }`
- `tools?: Array<{...}>` — tool definitions passed through to API
- `callClaudeExtended()` — returns `CallClaudeResult` with text, usage (including cache tokens), model, stopReason
- `callClaudeStream()` — return type changed from `Promise<string>` to `Promise<StreamResult>` with text + usage
- `callClaude()` unchanged (still `Promise<string>`)

### Schema (Supabase, project tappyckcteqgryjniwjg)

**chat_logs** — 14 columns: id, created_at, pill, input_text, attachments, response_text, model_used, input_tokens, output_tokens, cache_read_tokens, cache_creation_tokens, latency_ms, ip_hash, error

**ask_rate_limits** — 5 columns: ip_hash (PK), hourly_count, hourly_window_start, daily_count, daily_window_start

No RLS — service-role-only access via API routes.
