## Architecture

### Auto-classify (FIX 3)

When `pill: 'auto'` is sent to `/api/ask`:
1. Haiku classifier runs first (~400ms, `claude-haiku-4-5-20251001`, temperature 0)
2. Few-shot examples provide 4 exemplars (one per pill) in the messages array
3. Classifier returns `{"pill":"<category>","confidence":"high"|"low"}`
4. HIGH confidence: routes silently to resolved pill, sends `classified` SSE event
5. LOW confidence: returns a clarifier question in Milo voice, no main call

**Classifier prompt priority:** sales (drafting/writing) > publisher > buyer > vet (fallback for bare entity names)

**Key learning:** Haiku 4.5 with instruction-only prompts stubbornly classified everything as "vet." Adding few-shot examples in the messages array (4 user/assistant pairs) fixed routing immediately.

### Pill context boxes (FIX 2)

Persistent description below active pill, replacing D181 hover tooltips. Verbatim copy per directive:
- **Vet:** "Drop anything — a name, email, LinkedIn, screenshot. I check the blacklist first, then dig. You get red flags, green flags, and a confidence rating."
- **Sales:** "Buyer ghosted, pitching a vertical, re-engaging — tell me the situation and I draft the reply. The more context I have, the sharper the message."
- **Publisher:** "Publisher pitched you, you're recruiting, or something's off — I write the response or the outreach."
- **Buyer:** "Call prep, dispute responses, follow-ups, blacklist decisions — give me the context and I give you the move."

Small uppercase label above: "{PILL} MODE" (11px, weight 500, 0.14em tracking)

### Universal helper line (FIX 1)

"Pro tip: paste the whole conversation. Click in the chat, select all, paste here. The more context I have, the sharper I get."

12px, rgba(255,255,255,0.4), centered, always visible.

### Auto-run on vet drop (FIX 4)

When vet pill is active and a file is dropped:
- Immediately fires submit with "Vet this." + attached content
- No waiting for typed context (vet's input IS the entity)
- Other pills (sales/publisher/buyer): file attaches but does NOT auto-submit

### Blacklist hit visual tell (FIX 5)

API sends `{"type":"blacklist_hit"}` SSE event before response starts. Frontend renders the assistant message with `border-left: 2px solid rgba(239,68,68,0.6)` and `padding-left: 12px`.

### Copy audit (FIX 6)

All new copy verified:
- Zero "we"/"our"/"our AI" — all first-person Milo voice
- Zero emoji in UI chrome
- Clarifier question: "Is this someone to vet, or a situation where you want me to draft an outreach? I can go either way."

### Pills now toggleable

Pills are deselectable — clicking an active pill turns it off. When no pill is selected, queries go through auto-classify. This makes pills optional, not required.
